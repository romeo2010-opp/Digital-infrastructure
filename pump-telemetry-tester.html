Pump Telemetry Tester

This file is standalone and lives in the repo root so you can test the SmartLink pump monitoring and telemetry APIs directly from the browser. Station-side endpoints use the station bearer token, while internal telemetry endpoints use the internal bearer token.

Connection

Saved in your browser via local storage.

Helper POST /auth/login 
Use the station login form to fetch and fill the station bearer token automatically.
Helper POST /api/internal/auth/login 
Use the internal login form to fetch and fill the bearer token automatically.

Backend default in this repo is http://localhost:4000 unless you changed PORT.

Available APIs

These are the telemetry-related routes present in the repository.

Station Login Helper

Internal Login Helper

Station Monitoring

  • GET/api/monitoring/pumps/:pumpId
  • GET/api/monitoring/pumps/:pumpId/nozzles
  • POST/api/monitoring/telemetry/events
  • POST/api/dev/monitoring/simulate

Internal Telemetry

  • GET/api/internal/transactions/:transactionPublicId/pump-session
  • GET/api/internal/transactions/:transactionPublicId/telemetry
  • GET/api/internal/pump-sessions/:sessionReference/telemetry

Public Verification

  • GET/verify/receipts/:reference

Notes: monitoring reads require a station role such as MANAGER, ATTENDANT, or VIEWER. Telemetry event ingest and simulation require MANAGER or ATTENDANT. Internal telemetry endpoints require an internal token with support, finance, or risk view permission. Receipt verification is public and can return JSON or HTML depending on the Accept header.
